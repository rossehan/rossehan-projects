# ODD Order — 개인 발주서 탭 설계

## 요약
기존 ODD Order 앱(탭1: 공구 발주서, 탭2: 송장 변환, 탭3: 대시보드)에 **탭4: 개인 발주서**를 추가한다. 3PL(고고창고)에 위탁 출고를 요청하기 위해, 수취인 정보를 직접 입력하고 기존 발주서와 동일한 Excel 양식으로 다운로드하는 기능.

## 워크플로우
1. 폼에 수취인 정보 입력 → **추가** 버튼 클릭
2. 아래 테이블에 항목이 쌓임 (수정/삭제 가능)
3. **엑셀 다운로드** 클릭 → 기존 공구 발주서와 동일한 양식 출력
4. 다운로드 후 리스트 자동 초기화 (또는 수동 비우기)

## 입력 폼 필드

| 필드 | 필수 | 타입 | 기본값/비고 |
|---|---|---|---|
| 수취인명 | O | text | - |
| 핸드폰 | O | tel | - |
| 주소 | O | text | - |
| 상품 | O | select (드롭다운) | 기존 PRODUCT_MAP 7개 + DEFAULT_PRODUCT |
| 수량 | O | number | 1 |
| 배송메세지 | X | text | - |

## 발주서 Excel 출력 사양
- 기존 `xlsx-populate` 코드 재사용
- 시트1: '발주서' — 컬럼 A~P, 기존과 동일한 헤더/스타일
- 시트2: '참조' — 상품명+상품코드 7개
- 보내는분/연락처/주소 = `CONFIG.sender` (한아원)
- 상품명/상품코드 = PRODUCT_MAP에서 자동 매핑
- 상품주문번호: 빈 값 (스마트스토어 주문이 아니므로)
- 파일명: `개인발주서_YYYY-MM-DD.xlsx`

## 데이터 저장
- 저장 없음 (임시 리스트, 페이지 새로고침 시 초기화)
- 메모리 내 배열로 관리

## UI
- 탭4 버튼: 기존 탭 스타일과 동일
- 카드 레이아웃: 상단에 입력 폼, 하단에 리스트 테이블
- 테이블 각 행에 삭제 버튼
- 리스트가 비어있으면 다운로드 버튼 비활성화
- 공구 라운드 선택과 무관하게 독립 동작

## 범위 외
- 이메일 자동 발송 (수동으로 다운받은 파일을 이메일에 첨부)
- 네이버 API 연동 (이번 스코프에서는 제외)
- 수취인 주소록/프리셋 저장
