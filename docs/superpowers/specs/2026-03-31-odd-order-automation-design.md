# ODD 공구 발주서 자동화 - 디자인 스펙

**프로젝트명:** ODD Order Automation (공구 발주서 자동화)
**작성일:** 2026-03-31
**작성자:** rossehan + Claude

---

## 1. 개요

### 1.1 문제
ODD M-01 건강기능식품을 네이버 스마트스토어에서 공구(공동구매)로 판매할 때, 스마트스토어 주문 발주서와 3PL 창고(한아원/고고창고) 발주서 양식이 달라서 수동으로 엑셀을 변환해야 한다. 송장번호도 3PL에서 회신받은 후 스마트스토어 양식으로 다시 변환해서 업로드해야 한다.

### 1.2 해결
브라우저 기반 웹앱으로 엑셀 양식 변환을 자동화하고, 공구 현황 대시보드를 제공한다.

### 1.3 핵심 원칙
- 단일 HTML 파일, 서버/DB 없음
- 모든 데이터 처리는 브라우저에서 수행 (개인정보 서버 전송 없음)
- 공구 회차별 데이터는 localStorage에 보관

---

## 2. 워크플로우

```
1. [수동] 스마트스토어 관리자에서 주문 엑셀 다운로드
2. [자동] 웹앱에 업로드 → 3PL 한아원 발주서 양식으로 변환 + 다운로드
3. [반자동] Gmail 작성 창 자동 오픈 (받는사람/제목/본문 자동) → 발주서 첨부 후 전송
4. [수동] 한아원에서 송장번호 기재된 발주서 이메일 회신
5. [자동] 회신 엑셀을 웹앱에 업로드 → 스마트스토어 송장 업로드용 엑셀 생성
6. [수동] 스마트스토어에 송장 엑셀 업로드
```

## 3. 기능 상세

### 3.1 탭 1: 발주서 변환 (스마트스토어 → 3PL)

**입력:** 스마트스토어 주문 엑셀 (.xlsx)

**처리:**
1. 엑셀 파싱 후 옵션정보 → 3PL 상품명 + 상품고유코드 매핑
2. 수취인 정보 매핑 (수취인명, 연락처, 주소, 배송메세지)
3. 보내는분 고정값 채우기 (한아원)
4. (공란) 컬럼에 상품주문번호 삽입 (송장 매칭용)
5. 중복 주문 감지 (이미 등록된 상품주문번호 경고)
6. localStorage에 주문 데이터 저장 (대시보드용)

**3PL 발주서 컬럼 순서 및 매핑 (A열~P열):**

| 열 | 3PL 컬럼 | 소스 |
|---|---|---|
| A | 보내는분 | "한아원" (고정) |
| B | 보내는분 연락처 | "010-7701-2732" (고정) |
| C | 주소 (보내는분) | "서초대로 60길 18, 한아원 9층" (고정) |
| D | 번호 | 자동 순번 (1, 2, 3...) |
| E | 수취인명 | ← 스마트스토어: "수취인명" |
| F | 상품명 | ← "옵션정보"를 매핑 테이블로 변환 |
| G | 수량 | ← 스마트스토어: "수량" |
| H | 핸드폰 | ← 스마트스토어: "수취인연락처1" |
| I | 기타연락처 | ← 스마트스토어: "수취인연락처2" |
| J | 주소 (수취인) | ← 스마트스토어: "기본배송지" + " " + "상세배송지" |
| K | 배송메세지 | ← 스마트스토어: "배송메세지" |
| L | (공란) | ← 스마트스토어: "상품주문번호" (송장 매칭 키) |
| M | 상품고유코드 | ← 매핑 테이블에서 자동 (없으면 빈값) |
| N | 배송방식 | 빈값 (3PL이 채움) |
| O | 운송장번호 | 빈값 (3PL이 채움) |
| P | 택배사 | 빈값 (3PL이 채움) |

> 참고: 스마트스토어 엑셀 헤더명은 쌍따옴표 안의 문자열과 정확히 일치해야 함. 컬럼 찾기 시 trim() 처리 적용.

**출력:** `YYYY-MM-DD 한아원 발주서.xlsx` 다운로드

**Gmail 연동:**
- Gmail compose URL 자동 생성
- 받는사람: gogo@gogochango.com
- 제목: `[한아원] YY.MM.DD 한아원 발주서 전달의 건`
- 본문:
  ```
  안녕하세요,
  한아원 한로제입니다.

  YY.MM.DD 한아원 발주서 전달드립니다.
  확인 후 송장번호 공유 부탁드리겠습니다.


  감사합니다!
  한로제 드림
  ```
- 파일 첨부는 수동 (Gmail compose URL 제한)

**에러 처리:**
- 매핑 테이블에 없는 옵션정보 → 해당 행 빨간색 하이라이트 + "매핑 실패" 배지
- 중복 상품주문번호 → 경고 팝업 ("이미 등록된 주문 N건이 포함되어 있습니다")

### 3.2 탭 2: 송장 변환 (3PL → 스마트스토어)

**입력:** 3PL 회신 발주서 엑셀 (.xlsx) — 송장번호 포함

**처리:**
1. L열(공란) 컬럼에서 상품주문번호 추출
2. O열(운송장번호) 컬럼에서 송장번호 추출
3. 고정값 조합: 배송방법="택배,등기,소포", 택배사="CJ대한통운"
4. localStorage의 해당 주문에 송장번호 연결 + 발송상태 "완료" 업데이트

**출력 컬럼:**

| 컬럼 | 값 |
|---|---|
| 상품주문번호 | ← (공란) 컬럼 |
| 배송방법 | "택배,등기,소포" (고정) |
| 택배사 | "CJ대한통운" (고정) |
| 송장번호 | ← 운송장번호 컬럼 |

**출력:** `스마트스토어_송장업로드.xls` 다운로드

**부분 송장 처리:**
- 여러 번 송장 엑셀 업로드 가능 (한아원이 나눠서 보낼 경우)
- 대시보드에 누적 반영

### 3.3 탭 3: 대시보드

**공구 회차 관리:**
- 공구 생성 시 입력: 회차명, 시작일, 종료일, 인플루언서명 (선택)
- localStorage에 회차별 데이터 분리 저장
- 회차 탭으로 전환하여 과거 공구 데이터 조회 가능

**요약 카드:**
- 총 주문 수
- 발송 완료 건수
- 발송 대기 건수
- 총 매출액

**옵션별 매출 분석:**
- 옵션 필터 칩 (전체, 5일분 1개, 5일분 3개, ...)으로 선택 가능
- 테이블: 옵션명, 단가, 주문 수, 금액
- 합계 행

**금액 계산:**
- 스마트스토어 엑셀의 "최종 상품별 총 주문금액" 컬럼 사용
- 공구별 가격이 다를 수 있으므로 실제 결제 금액 기준

**주문 검색 (CS 대응):**
- 수취인명 또는 연락처로 검색
- 검색 결과: 수취인명, 연락처, 옵션, 주소, 발송상태, 송장번호

**엑셀 다운로드:**
- 전체 주문 엑셀 다운로드 (수취인, 옵션, 금액, 송장번호, 발송상태 포함)
- 옵션별 매출 엑셀 다운로드

## 4. 상품 매핑 테이블

**매핑 규칙:** 옵션정보 문자열을 아래 순서대로 매칭 (긴 문자열 우선, 첫 번째 매칭에서 중단)

| 매칭 우선순위 | 스마트스토어 옵션정보 (contains) | 3PL 상품명 | 상품고유코드 | 현재 판매가 |
|---|---|---|---|---|
| 1 | 30일분 3개 + 5일분 4개(110개입) | 리필팩 3개 + 스타터키트 4개 | (빈값) | 350,000원 |
| 2 | 30일분 2개 + 5일분 3개(75개입) | 리필팩 2개 + 스타터키트 3개 | (빈값) | 240,000원 |
| 3 | 30일분 1개 + 5일분 2개(40개입) | 리필팩 1개 + 스타터키트 2개 | (빈값) | 130,000원 |
| 4 | 5일분 3개(15개입) | 스타터키트 5일분 3개 | (빈값) | 65,000원 |
| 5 | 5일분 1개 | 오드 M-01 스타터키트 5일분 | P00300280 | 26,000원 |

> - 긴 문자열부터 매칭하여 "5일분 1개"가 "30일분 1개 + 5일분 2개"에 잘못 매칭되는 문제 방지
> - 상품고유코드가 (빈값)인 세트 상품은 3PL 발주서에서 해당 컬럼을 비워둠
> - 판매가는 공구마다 변동 가능. 대시보드 금액은 스마트스토어 엑셀의 "최종 상품별 총 주문금액" 컬럼 사용

## 5. 데이터 저장 (localStorage)

```json
// localStorage key: "odd-orders"
{
  "activeRoundId": "round-1",
  "rounds": [
    {
      "id": "round-1",
      "name": "1차 공구",
      "startDate": "2026-03-28",
      "endDate": "2026-04-03",
      "influencer": "인플루언서A",
      "orders": [
        {
          "productOrderNo": "2024030012345",
          "recipientName": "김철수",
          "phone": "010-1234-5678",
          "option": "5일분 1개",
          "threePLProductName": "오드 M-01 스타터키트 5일분",
          "quantity": 1,
          "amount": 26000,
          "address": "서울시 강남구 테헤란로 123",
          "status": "shipped",
          "trackingNo": "123456789012",
          "orderedAt": "2026-03-28",
          "shippedAt": "2026-03-29"
        }
      ]
    }
  ]
}
```

**스키마 보충:**
- `activeRoundId`: 현재 활성 공구 회차 ID. 탭 1/2 상단에 표시되며 변경 가능.
- `orderedAt`: 스마트스토어 엑셀의 "주문일시" 컬럼에서 추출
- `shippedAt`: 송장 변환 시점의 날짜
- `status`: "pending" (발송대기) 또는 "shipped" (발송완료)
- 중복 상품주문번호 업로드 시: 기존 건을 최신 데이터로 덮어씀 (중복 방지)

## 9. 공구 회차 UX

- 탭 1(발주서 변환), 탭 2(송장 변환) 상단에 **현재 활성 공구 회차** 표시
- 드롭다운으로 회차 변경 가능
- 새 공구 생성 시 활성 공구가 자동 전환됨
- 업로드한 데이터는 현재 활성 공구 회차에 저장됨
- **첫 사용 시:** 공구 회차가 없으면 "새 공구 만들기" 안내를 표시하고, 공구를 생성해야 탭 1/2 사용 가능

---

## 9.1 주문 상태 필터링

- 스마트스토어 엑셀의 "주문상태" 컬럼 확인
- "결제완료" 또는 "발주확인" 상태인 주문만 3PL 발주서에 포함
- 취소/환불/반품 상태 주문은 자동 제외하고, 제외 건수를 화면에 표시
  - 예: "총 300건 중 취소 5건 제외, 295건 변환됨"

## 10. UX 상세

### 10.1 빈 상태 (Empty State)
- **탭 1/2 — 공구 미생성 시:** "먼저 공구를 만들어주세요" + 새 공구 만들기 버튼
- **탭 1/2 — 업로드 전:** 업로드 영역만 표시, 미리보기 테이블은 숨김
- **탭 3 — 데이터 없음:** "아직 주문 데이터가 없습니다. 발주서를 변환하면 여기에 현황이 표시됩니다"

### 10.2 작업 완료 피드백 (토스트 메시지)
- 변환 성공: "N건 변환 완료!" (취소 건 제외 시 "총 300건 중 취소 5건 제외, 295건 변환됨")
- 다운로드 완료: "YYYY-MM-DD 한아원 발주서.xlsx 다운로드 완료"
- 송장 반영: "N건 송장 반영 완료! 발송완료율 X%"
- 토스트는 3초 후 자동 사라짐

### 10.3 활성 공구 배너
- 모든 탭 상단에 고정 배너: "현재: 1차 공구 (26.03.28~26.04.03 | 인플루언서A)" + 변경 버튼
- 잘못된 공구에 데이터를 넣는 실수 방지

### 10.4 워크플로우 진행 표시기
- 탭 1/2 상단에 스텝 표시: `엑셀 업로드 → 변환 확인 → 다운로드 → Gmail 발송`
- 현재 단계 하이라이트, 완료된 단계는 체크 표시
- 사용자가 다음에 뭘 해야 하는지 바로 알 수 있도록

### 10.5 데이터 백업/복원
- 대시보드 하단에 "데이터 백업 (JSON)" 다운로드 버튼
- "데이터 복원" 업로드 버튼 (JSON 파일 선택)
- localStorage 전체 데이터를 JSON으로 내보내기/가져오기
- 브라우저 데이터 삭제 시 복원 가능
- 다른 PC에서 이어서 작업 가능

### 10.6 실행 취소 (Undo)
- 엑셀 업로드로 localStorage 데이터가 변경되기 직전에 스냅샷 자동 저장
- "되돌리기" 버튼으로 직전 상태 복원
- 잘못된 파일 업로드 시 데이터 오염 방지

### 10.7 대시보드 차트 시각화
- 옵션별 매출 비중: 도넛 차트 또는 가로 막대 차트
- CSS만으로 구현 (라이브러리 없음)
- 필터 칩 선택 시 차트도 연동하여 갱신
